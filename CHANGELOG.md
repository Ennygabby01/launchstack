# Changelog

All notable changes to LaunchStack are documented here.

---

## [1.1.5] - 2026-03-14

### Improved

- All backend templates improved with production-ready structure
  - **Express**: added `src/config/env.js` (startup env validation), `nodemon.json`, and versioned `src/routes/v1/` pattern
  - **Fastify**: added `src/config/env.js`, `setErrorHandler` for consistent error responses, `src/plugins/` folder
  - **Django**: `settings.py` now reads `DATABASE_URL` via `dj-database-url` (falls back to SQLite); added `app/wsgi.py`; `requirements.txt` now includes `dj-database-url`, `psycopg2-binary`, and `gunicorn`
  - **NestJS**: added `src/config/config.ts` and `src/common/guards/` placeholder
- All modules now include NestJS-compatible TypeScript files (`.module.ts`, `.service.ts`, `.middleware.ts`) alongside existing JS files
- `prisma/schema.prisma` is now auto-generated when Prisma is selected â€” ready for `npx prisma generate`
- `.env.example` improvements: section headers, `SHADOW_DATABASE_URL` (Prisma + PostgreSQL), `REDIS_PASSWORD`, `LOG_LEVEL`, `STRIPE_PRICE_BASIC` / `STRIPE_PRICE_PRO`
- `README.md` (generated) quick start now uses `npm run start:dev` for NestJS, and correct `python` + `venv` instructions for Django
- Django projects now get `SECRET_KEY`, `DEBUG`, and `ALLOWED_HOSTS` in their `.env.example` instead of `NODE_ENV`/`PORT`
- Docker `Dockerfile`: `CMD` changed from hardcoded `node src/index.js` to `npm start`; deprecated `--only=production` replaced with `--omit=dev`
- Next.js template: fixed branding (`"Generated by Launchstack"`), added `next.config.js` with `reactStrictMode`, added `public/` directory
- React Vite template: added `src/App.css` with a minimal CSS reset, auto-imported in `App.jsx`

## [1.1.2] - 2026-03-14

### Fixed
- Removed `@nestjs/cli` from NestJS dev dependencies â€” it pulled in `webpack`, `@angular-devkit/*`, and `@electric-sql/pglite` (100MB+), causing installs to hang or fail. Users who need the NestJS CLI can install it globally: `npm i -g @nestjs/cli`

### Added
- `launchstack-templates` repo is now live at [github.com/Ennygabby01/launchstack-templates](https://github.com/Ennygabby01/launchstack-templates) â€” template updates can now be pulled independently of CLI releases

## [1.1.1] - 2026-03-14

### Fixed
- GitHub AI mode now uses the correct GitHub Models API endpoint (`models.inference.ai.azure.com`) instead of the Copilot-specific endpoint â€” regular GitHub PATs now work correctly
- AI error messages now include the response body for easier debugging

### Improved
- Ollama detection now checks MacPorts (`/opt/local/bin`), Homebrew Apple Silicon (`/opt/homebrew/bin`), and Intel/manual installs (`/usr/local/bin`) when `ollama` is not found in PATH
- GitHub Models is now the recommended AI provider (faster than Ollama, free tier available)
- README clarifies that `GITHUB_TOKEN` should be exported per session, not stored permanently â€” tokens expire

### Docs
- Added MacPorts install instructions for Ollama
- Added guidance on token expiry and session-scoped export

## [1.1.0] - 2026-03-13

### Added
- AI-Assisted Mode: describe your app in plain English and get a full stack configuration suggested automatically
- Ollama support (local AI, no API key required)
- GitHub Models API support (requires `GITHUB_TOKEN`)
- Freshness guard: warns before installing packages not updated in over 12 months
- Deprecated package detection with modern replacements suggested
- `launchstack doctor` command: environment check + template dependency freshness scan
- `launchstack templates update` / `launchstack templates list` commands
- Docker setup generation when Docker is detected
- Package manager selection: npm, pnpm, or yarn
